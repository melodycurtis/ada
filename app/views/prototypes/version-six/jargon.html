Jargon buster

Term Definition
ADA Analytical Data Access. The internal DfE service that allows staff to find data and access analytical tools.
ADF Azure Data Factory. Azure Data Factory is a managed cloud service that's built for complex hybrid extract-transform-load (ETL), extract-load-transform (ELT), and data integration projects.
ADLS Azure Data Lake Storage is a cloud-based, enterprise data lake solution. It's engineered to store massive amounts of data in any format, and to facilitate big data analytical workloads. You use it to capture data of any type and ingestion speed in a single location for easy access and analysis using frameworks.
Analyst workbench The toolkit that allows analysts to perform analysis and modelling of data. This is the internal functional name for Databricks, a commercially licensed cloud-based platform.
Apache spark An open-source, distributed processing system used to power big data workloads and complex processing to enable fast execution of analytic queries against data of any size.
API An Application Programming Interface (API) is a communication interface to allow two or more applications to communicate. Often used by organisations as a method for external users to retrieve or submit data and/or issue commands to servers.
Azure cloud services Azure cloud services is a suite of cloud based services that Microsoft offer. This covers a broad range of infrastructure including Data Storage, Computer resources and other higher level services built on these foundation services.
Cluster A set of computation resources and configurations where you run data engineering, data science and data analytics workloads. It is the engine that drives the notebook that allows you to perform your analysis. Before you start using notebooks, Databricks will prompt you to attach a cluster. A cluster must be running to view data in the data tab.
Data discovery platform The name for the DfE service that allows you to search, discover and request access to data across departments. This includes "Data Marketplace" to view and request access to data and "Data Governance" for further information on the data held . The team that look after this product are called Data Governance or Metadata Team. It is built using Axon and Informatica licensed products. You can access this from the "Find Data" tile on the ADA homepage.
Data sets A collection of related sets of information. For example - a dataset about pupil attendance which includes pupil, school, and attendance information sets.
Databricks A cloud based data engineering and analytics platform which DfE has licensed. A suite of tools to support data engineers and analysts to store data and tooling - to query and analyse large data sets.
Delta file The format of the way data is stored in the cloud-based storage delta lake. A delta file compresses information and only makes changes to the parts of the file that have changed when you need to copy, download or upload it.
Delta lake A storage layer that provides the foundation for storing data and table in the Databricks platform. Delta lake is open source software and it is the default storage format for all operations on Databricks.
Machine learning Machine learning is a field devoted to understanding and building methods that let machines "learn". Machine learning algorithms build a model based on sample data, known as training data, in order to make predictions or decisions without being explicitly programmed to do so. Databricks Machine Learning provides an integrated machine learning environment that helps you simplify and standardise your ML development processes.
Metadata A set of data that describes and information about other data. For example school metadata might include the school name, identifier number, address.
MVP Minimum viable product. The first iteration of a product or service. It will have enough features to allow a user to achieve their goal but may still have some limitations which will be delivered over time.
Notebook Databricks notebooks provide real-time coauthoring in multiple languages (R, SQL, Python), automatic versioning, and built-in data visualisations. A tool for developing code and presenting results.
Parquet file Apache Parquet is a file format used by data engineers. It offers flexibility, scalability and is open-source. In addition to data, it contains metadata including schema and structure. Each file stores the data and the standards used for accessing each record - making it easier to decouple services that write, store, and read parquet files.
Private beta The stage of delivery in a project or service where the service is available to a limited number of users - so that it can be thoroughly tested and improved until it is released to a wider audience. This is part of the gov.uk Service Standard.
Python A programming or coding language which the analyst community use to develop their models for analysis. It is widely used in data analysis and data science for importing, analysing and visualisation of data.
R A programming or coding language and environment which the analyst community use to develop their models for analysis. It has features which are useful for statistical analysis and graphical representation.
RShiny A package to build interactive web applications straight from R. You might use this to build dashboards where you want to show metrics in an interactive way.
R Studio A platform, or in technical terms an Integrated Development Environment (IDE), where users can write code using R to build and share work. It is available in two formats: RStudio Desktop is a regular desktop application while RStudio Server runs on a remote server and allows accessing RStudio using a web browser.
Reports and dashboards A feature in the ADA service that provides a library of reports and dashboards that can be used by DfE staff to find and query data. In some cases - access will be restricted to authorised users only. The ambition is to grow this library over time. This area also provides navigation to the Explore Education Statistics public-facing portal.
Scalability The ability to quickly increase or decrease the size or power of an IT solution or resource.
Scaling in / out Normally done when scaling up can't provide enough resources for the workloads. With scaling out, resources or data is split into several databases - across servers, and each can be scaled up or down independently.
Scaling up / down Useful to handle spikes in workloads where the performance level can't satisfy the demands. Scaling up lets you add more resources to handle peak workloads. When the resources are not needed - scaling down lets you go back to the original state and save on cloud costs.
SQL Structure Query Language (SQL) is a programming or coding language for use in data and database manipulation.
Tech spike Development teams will carry out a technical (tech) spike within a set period of time (timeboxing) to explore, research possible solutions to delivery a feature or functionality needed. This may involve some writing of code, but typically there is no commitment to delivering the functionality.
