Using R, Python and SQL in Databricks

How clusters work

A cluster is a group of remote machines providing processing power.
They help to manage the data stored in the platform, so even if you don't use Databricks to run your analyses, you will need to use clusters to access the data.

You will be given access to clusters when you request access to the platform.
If you think you should have access to a cluster that isn't listed in your account - contact the ADA project team ADAPTTeam@education.gov.uk



Available clusters

  1. Click on the Compute button on the left side menu to see the clusters available to you.
    Click on each cluster to see more details.
    Click on the play icon at the end of the row - to start the cluster.


  2. In the DfE we use two types of clusters.

    Shared Access Clusters

    These clusters are linked to specific projects to help manage usage.
    They are set up by an administrator and will be sized for the project needs.
    They will scale up and down to manage varying workloads.
    You cannot use a shared cluster if you are working in R.

    Single User Access (Personal) Clusters

    These clusters are linked to a single user and the size will be limited.
    They are useful when you need to make configuration changes that are specific to your needs.
    Use this cluster if you are working in R.
