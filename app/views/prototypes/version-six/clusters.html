Using R, Python and SQL in Databricks

How clusters work

A cluster is a group of machines that provide processing power.

If you don't use Databricks to run your analyses -you will still need to use clusters to access the data.

You will be given access to shared clusters when you get access to a dataset.


Available clusters

  1. To see information about the clusters available to you - click on the Compute button on the left menu.
    Click on each cluster to see more details.
    Click on the play icon at the end of the row - to start the cluster.


    2. If you think you should have access to a cluster that isn't listed - contact the ADA project team ADAPTTeam@education.gov.uk


Starting a clusters

If you try to run R code on a shared cluster you will get this error message.

Your administrator has only allowed sql and python commands on this cluster. This execution contained at least one disallowed language.

Creating a cluster

Databricks administrators will create shared project clusters.
You can can create personal clusters in line with prepared templates known as policies.
If you're using R you need to create one of these personal clusters.

The most basic policy is the DfE Personal Compute

This allows you to create a small cluster for personal use. All users have access to this policy.

You can create a cluster using this policy by clicking the Create Personal Compute button.
