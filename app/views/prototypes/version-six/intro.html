Getting started with Databricks

How Databricks works

Login

When you're given access to Databricks you'll get an email with instructions to register with multi-factor authentication (MFA) for the “CIP Azure portal and DevOps”.

CIP is the "Cloud Infrastructure Platform" and Azure is the product name for the environment where Databricks is hosted.

You'll also get an email from our project team to confirm you've been set up and have access to your data.

You'll need to log in each time.

Left menu

Understand how the left menu works.

Workspaces

A space for accessing your assets. It organises objects - notebooks, libraries, dashboards, and experiments into folders and provides access to data objects and computational resources.

In the DfE we have different workspaces with different purposes - develoment, test and production.

Clusters

A set of computation resources where you can run notebooks and jobs. There are two types of clusters: all-purpose and job.

Before you get started using notebooks, you will be prompted to attach a cluster. A cluster must be running to view data in the data tab.

Unity catalog

Access to data is governed by the Unity Catalog solution in Databricks. Data is organised in the following hierarchy:

A place to administer data access that apply across all workspaces.

In Unity Catalog, the hierarchy of primary data objects flows from metastore to table:


Metastore: The top-level container for metadata. Each metastore uses a three-level name (catalog.schema.table) that organizes your data.

Example: preprod_catalog_p02.ada_demo.csc_data

Catalog: The first layer of the object hierarchy, used to organize your data assets.

Schema: Also known as databases, schemas are the second layer of the object hierarchy and contain tables and views.

Table: At the lowest level are tables and views.

Notebooks

Notebooks are used for creating code and presenting results.

A notebook allows you to write code a section at a time, using cells. It can have one or more cells.


Using mixed languages

Each cell can contain a different langauge. One could be R, another could be SQL.

Cells use the default language of the notebook but you can prime cells for different programming languages using magic commands. Magic commands allow you to switch between Python, SQL and R.
