Getting started with Databricks

Concepts you need to know

Databricks uses three environments:
1. Data Science & Engineering
2. Machine Learning
3. SQL

Left menu

The left menu expands when you hover over it.

Use the menu to select the enviroment you need to work in.

You can set it to expand or collape permanently using the Menu options at the bottom.

Or leave it on Auto if you want to hover.

Notebook

Notebooks are used for creating data science and machine learning workflows and collaborating with colleagues. They provide real-time coauthoring in multiple languages, automatic versioning, and built-in data visualizations.

Workspace

An space for accessing your assets. It organises objects - notebooks, libraries, dashboards, and experiments into folders and provides access to data objects and computational resources.

Cluster

A set of computation resources and configurations where you can run notebooks and jobs. There are two types of clusters: all-purpose and job.

Unity catalog

A place to administer data access that apply across all workspaces.

In Unity Catalog, the hierarchy of primary data objects flows from metastore to table:


Metastore: The top-level container for metadata. Each metastore uses a three-level name (catalog.schema.table) that organizes your data.

Catalog: The first layer of the object hierarchy, used to organize your data assets.

Schema: Also known as databases, schemas are the second layer of the object hierarchy and contain tables and views.

Table: At the lowest level in the object hierarchy are tables and views.
